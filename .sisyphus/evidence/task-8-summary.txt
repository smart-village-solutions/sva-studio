## Key Features Implemented

### Capacity Management
- Max concurrent previews: 10 (pilot phase)
- Queue system: 5 slots with FIFO + wait time notifications
- Priority system: Label-based (high/default) with budget enforcement

### Cost Control
- Monitoring-first approach (no blind caps)
- Progressive evaluation: Day 30 → Day 90 → Regular reviews
- Metrics: `preview_cost_estimated_eur`, `preview_budget_utilization_percentage`

### Idle Cleanup
- 3-stage lifecycle: Active (0-6d) → Stale (7d) → Destroy (14d)
- User control: `/preview keep` command with 2x extension limit
- Robust error handling: Retry + Escalation + SLA

### Monitoring Integration
- 6 Prometheus metrics defined
- 3 alerts configured (capacity, cleanup failure, budget)
- GitHub automation (labels, comments at each stage)
