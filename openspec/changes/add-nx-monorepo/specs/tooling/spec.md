## ADDED Requirements
### Requirement: Nx Workspace Initialized at Repo Root
The project SHALL provide an Nx integrated workspace configured at the repository root using npm, without creating nested subdirectories.

#### Scenario: Workspace initialized in place
- **WHEN** the workspace is generated
- **THEN** `nx.json`, `package.json`, and workspace config files reside at the repository root
- **AND** no extra top-level app folder (e.g., `sva-studio/`) is created
- **AND** Nx Cloud is disabled

### Requirement: Preserve Existing Tooling Conventions
The project SHALL keep existing formatting and commit conventions while adding Nx tooling.

#### Scenario: Prettier intact
- **WHEN** Nx tooling is added
- **THEN** existing Prettier configuration remains effective and is not overwritten

#### Scenario: Conventional commits unaffected
- **WHEN** Nx scripts and generators are added
- **THEN** commit conventions (Conventional Commits) remain unchanged

### Requirement: Provide Baseline Scripts
The workspace SHALL expose baseline scripts for linting and testing provided by the Nx TypeScript preset.

#### Scenario: Lint/test scripts available
- **WHEN** the workspace is initialized
- **THEN** `package.json` contains runnable scripts for linting and testing generated by Nx
- **AND** they succeed on a clean checkout after dependency installation
