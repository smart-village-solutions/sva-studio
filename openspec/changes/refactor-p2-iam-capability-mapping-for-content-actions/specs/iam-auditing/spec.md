## ADDED Requirements

### Requirement: Audit-Einträge können Capability-Mapping nachvollziehen

Das System SHALL Audit- oder Diagnosepfade so gestalten, dass fachliche Aktionen und ihr Mapping auf primitive Rechte nachvollziehbar bleiben.

#### Scenario: Deny- oder Allow-Entscheidung bleibt erklärbar

- **WHEN** eine fachliche Inhaltsaktion erlaubt oder verweigert wird
- **THEN** kann die Entscheidung auf die zugrunde liegenden primitiven Rechte zurückgeführt werden
- **AND** Audit- oder Diagnosepfade verlieren die fachliche Lesbarkeit nicht
